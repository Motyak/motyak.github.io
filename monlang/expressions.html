which the interpreter can evaluate for a value

operation:

- built from two expressions
- makes an expression grow horizontally
- order of evaluation determined by
  precedence and associativity rules
- the use of parentheses may change the order
  of evaluation

1 + 2
1 + 2 + 3
1 + 2 * 3
(1 + 2) * 3
(2 ** 3) ** 2
[2, 4, 6] |> map(times2)
5 in 1 .. 10 || ERR("somethin' wrong")


block expression:

- built from statements
- makes an expression grow vertically
- as within a Program, statements are evaluated
  from top to bottom
- the trailing statement, if producing a value,
  is used to return as the final result
  of the all block expression

{
    var a 1 + 2
    var b 3
    a * b
}
